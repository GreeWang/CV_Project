
# Project 3 实施计划

## 1. 项目目标

本项目的目标是构建一套完整的视频目标移除与修复（inpainting）流水线。
给定一段包含行人、自行车或车辆等动态目标的视频，系统应能够：

1. 自动检测并分割动态目标。
2. 区分真正运动的目标和静止目标。
3. 在跨帧情况下生成时间一致的 mask。
4. 移除选定目标并恢复背景。
5. 使用定量指标和定性对比评估方法效果。

本项目应按作业要求组织为三个部分：

1. Part 1：经典 baseline 流水线
2. Part 2：SOTA 复现流水线
3. Part 3：一个优化或扩展尝试

---

## 2. 交付内容

最终提交必须包括：

1. `公开 GitHub 仓库`
   仓库中应包含源代码、依赖安装说明、使用示例、模型权重信息，以及 README 中的可视化结果。
2. `PDF 报告`
   使用 CVPR LaTeX 模板。报告正文长度应为 6 到 8 页，不含参考文献。
3. `演示视频`
   所有必做数据集的处理后视频都需要导出，并打包为 `videos.zip`。

---

## 3. 推荐的整体策略

为了降低风险并让项目更易于管理，建议按照渐进式三阶段策略实现：

1. `Part 1`
   使用目标分割、运动分析和传统修复方法搭建经典 baseline。
2. `Part 2`
   用更强的视频分割和视频修复方法替换经典模块。
3. `Part 3`
   围绕 mask 质量、时间一致性或复杂背景重建，增加一个重点改进项。

推荐的实际组合：

1. `Part 1`
   `YOLOv8-Seg + Lucas-Kanade 稀疏光流 + mask 膨胀 + cv2.inpaint + 时间背景复制`
2. `Part 2`
   `SAM 2 + ProPainter`
3. `Part 3`
   `Mask 精修与时间一致性优化`

推荐这个组合的原因：

1. YOLOv8-Seg 相对容易部署，适合作为快速 baseline。
2. 与 VGGT4D 或 SAM 3 这类非常新的方法相比，SAM 2 更适合作为可落地的复现目标。
3. ProPainter 在项目说明中被明确推荐，而且其视频修复能力较强。
4. Mask refinement 作为 Part 3 的方向，比直接做完整生成式流水线更稳妥。

## 4. 系统拆解

项目应拆分为以下模块：

1. `数据准备`
   整理视频、抽取帧、调整分辨率，并管理输入/输出文件夹。
2. `分割`
   为每一帧生成候选目标 mask。
3. `运动分析`
   判断某个目标是动态还是静态。
4. `Mask 后处理`
   执行膨胀、孔洞填充、时间平滑和一致性 refinement。
5. `修复`
   进行目标移除与背景恢复。
6. `评估`
   在有 ground truth 时计算 JM、JR、PSNR 和 SSIM。
7. `可视化`
   保存并排对比结果、mask overlay 和关键帧比较图。
8. `实验管理`
   组织配置、日志、输出结果和 checkpoint。

## 6. Part 1 计划：经典 Baseline

### 6.1 目标

基于经典计算机视觉逻辑构建完整的目标移除流水线：

1. 检测候选目标
2. 判断其是否为动态目标
3. 生成 mask
4. 使用传统修复方法将其移除

### 6.2 分割模块

推荐模型：

1. `YOLOv8-Seg`

可选替代：

1. `Mask R-CNN`

建议关注的动态目标类别：

1. person
2. bicycle
3. motorcycle
4. car
5. bus
6. truck

每帧输出内容：

1. bounding boxes
2. instance masks
3. class labels
4. confidence scores

### 6.3 动态目标判定

仅做检测还不够，因为有些目标虽然存在，但可能是静止的。
为了避免误删静态目标，需要增加一个基于运动的过滤阶段。

实现方式：

1. 在每个实例 mask 内检测特征点。
2. 使用 Lucas-Kanade 稀疏光流在相邻帧之间跟踪这些点。
3. 计算运动幅度统计量，例如平均位移或中位位移。
4. 仅保留运动超过阈值的目标。

建议改进：

1. `全局运动补偿`
   如果相机在运动，需要先估计背景运动，再计算相对目标运动。
2. `时间持续性约束`
   要求某个目标在连续若干帧中都被判定为动态后，才执行移除。

### 6.4 Mask 后处理

对 mask 做后处理，使其更适合修复：

1. 二值膨胀
2. 孔洞填充
3. 去除过小的连通域
4. 跨帧时间平滑

目的：

1. 覆盖运动模糊和边界误差
2. 减少时间闪烁
3. 提升修复质量

### 6.5 修复模块

基础实现：

1. `cv2.inpaint` 的 Telea 方法
2. `cv2.inpaint` 的 Navier-Stokes 方法

改进实现：

1. 先在相邻帧中寻找同一位置的干净背景像素
2. 如果有可用的时间信息，则直接复制或聚合这些像素
3. 如果没有可用的时间背景信息，则回退到空间修复

建议策略：

1. `优先做时间复制`
2. `再用空间修复兜底`

### 6.6 Part 1 的预期结果

这一部分在以下场景中应能较好工作：

1. 目标运动较有限
2. 背景相对静止
3. mask 面积适中

可能的弱点：

1. 背景重建模糊
2. 无法很好处理大面积遮挡
3. 在拥挤场景中 mask 不稳定
4. 在相机运动时容易失败

## 7. Part 2 计划：SOTA 复现

### 7.1 目标

将经典 baseline 替换为现代 AI 驱动流水线，以实现：

1. 更高质量的 mask
2. 更好的时间一致性
3. 在大面积遮挡区域下更好的修复效果

### 7.2 推荐方法选择

项目要求从提供的方法中选择一个动态 mask 提取方法。

推荐选择：

1. `SAM 2`

备选方案：

1. `Track Anything`

为什么 SAM 2 是较好的选择：

1. 它性能强，而且讨论度高
2. 它支持视频分割
3. 它更容易在报告中进行论证
4. 相比更新但尚不成熟的方法，它更实用

### 7.3 使用 SAM 2 生成 Mask

实现步骤：

1. 搭建官方环境，并确认 demo 可以运行
2. 将视频帧整理为所需格式
3. 通过 prompt 初始化目标
4. 将分割结果传播到整段视频
5. 为每一帧导出二值 mask

Prompt 选项：

1. `手动 prompt`
   先使用手动框或点来初始化，确保先得到可运行 baseline。
2. `自动 prompt`
   使用 YOLO 检测框自动生成 prompt boxes。

推荐计划：

1. 先完成手动 prompt 版本
2. 若时间允许，再加入自动 prompt 生成，作为工程改进点

### 7.4 使用 ProPainter 做修复

实现步骤：

1. 搭建官方环境
2. 先运行官方 demo
3. 查看所需输入结构
4. 将你的 mask 转换成所需格式
5. 在你自己的视频和 mask 上运行 ProPainter

预期输入：

1. 视频帧
2. 与帧对齐的二值 mask

预期输出：

1. 修复后的帧
2. 最终修复视频

### 7.5 Part 2 的成功标准

Part 2 应当在以下方面明显优于 Part 1：

1. mask 质量
2. 时间一致性
3. 大面积遮挡区域的恢复质量
4. 背景纹理保留能力

## 8. Part 3 计划：优化与扩展

### 8.1 目标

找出 baseline 或 SOTA 流水线中的一个局限，并尝试改进。

### 8.2 推荐方向

最稳妥、最实际的选择是：

1. `Mask refinement 与时间一致性增强`

推荐原因：

1. 工程风险较低
2. 能自然融入 Part 2
3. 容易做出肉眼可见的效果提升
4. 在报告中也容易分析

### 8.3 可选实现方案

#### 方案 A：Mask refinement

1. 精修粗糙边缘
2. 扩大不稳定边界区域
3. 利用相邻帧对 mask 做时间平滑
4. 融合多个相邻帧的 mask

#### 方案 B：更好的 prompt 生成

1. 使用 YOLO 检测框作为 SAM 2 的自动 prompt
2. 将运动线索与检测 prompt 结合
3. 提升长视频初始化效果

#### 方案 C：针对困难区域的生成式修复

1. 找出背景从未出现过的区域
2. 使用生成式图像模型对关键帧进行修复
3. 将修复内容传播到邻近帧

这个方向更有挑战性，但风险也更高。

### 8.4 Part 3 的最低可接受结果

即使改进效果不强，Part 3 也应至少包含：

1. 明确描述优化思路
2. 完成一次实现尝试
3. 有前后对比结果
4. 分析它为什么有效或为什么失败

## 9. 数据集计划

### 9.1 必做数据集

项目必须包含：

1. `Wild Video`
   自己拍摄或生成的一段包含动态目标的视频
2. `bmx-trees`
3. `tennis`

### 9.2 可选但推荐的数据集

1. `DAVIS`

DAVIS 的价值在于：

1. 它是研究中常用的标准数据集
2. 它更方便做评估
3. 它有助于给出更强的定量结果

### 9.3 Wild Video 采集建议

为降低风险，建议至少录制两段视频：

1. `简单场景`
   一个运动目标、背景稳定、时长较短
2. `困难场景`
   多个运动目标或更严重的遮挡

实际建议：

1. 时长控制在 5 到 10 秒
2. 避免剧烈相机运动
3. 控制目标数量适中
4. 如果算力有限，使用 720p 或更低分辨率

## 10. 评估计划

### 10.1 Mask 指标

作业要求包括：

1. `JM`
   mean IoU
2. `JR`
   IoU recall

这些指标应用于：

1. Part 1 的 masks
2. Part 2 的 masks
3. 如果有改进，也评估 Part 3 的 masks

### 10.2 视频质量指标

作业要求包括：

1. `PSNR`
2. `SSIM`

重要说明：

1. 只有在存在干净 ground truth 帧时，才能计算这些指标
2. 对于没有 ground truth 的 wild videos，只能做定性比较

### 10.3 定性评估

建议准备以下可视化对比：

1. 输入帧
2. 预测 mask
3. Part 1 结果
4. Part 2 结果
5. Part 3 结果
6. 局部放大对比
7. 失败案例

每个场景至少选取：

1. 一帧目标刚进入画面的时刻
2. 一帧发生主要遮挡的时刻
3. 一帧目标离开后的时刻

## 17. 风险与备选方案

### 主要风险

1. SOTA 环境可能难以搭建。
2. 某些数据集可能无法为所有指标提供合适的 ground truth。
3. Wild videos 可能过于复杂、不稳定。
4. Part 3 可能耗时过多。

### 备选方案

如果 `SAM 2` 太难：

1. 改用 `Track Anything`
2. 或保留更强的工程化流水线，例如基于 YOLO prompt 和 tracking 的方案

如果 `ProPainter` 太难：

1. 改用 `E2FGVI`
2. 同时保证 Part 1 已完整实现，以确保项目整体仍然完整

如果 `Part 3` 效果较弱：

1. 保留实现尝试
2. 强调失败分析与局限性
3. 解释为什么该改进没有泛化成功

如果 `ground truth` 不足：

1. 用 DAVIS 做标准定量实验
2. 将必做数据集主要用于定性结果展示
